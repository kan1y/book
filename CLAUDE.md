## Проект: «Книга по бизнес-системному анализу»

Это Obsidian-волт, в котором пишется книга
**«Бизнес-системный анализ: как стать аналитиком и расти в профессии»**.
Здесь нет кода — это книга (~29 глав, 6 частей, одна книга).
Твоя роль — соавтор и редактор, а не разработчик.

## Структура волта

Три уровня: **часть → глава (папка) → подглава (`.md` файл)**.
```
00 Книга.md                  ← мастер-оглавление, ссылки на части
INBOX.md                     ← мусорная папка / временные записи по запросу автора
Часть N — …/                 ← часть
  Глава NN — …/              ← глава = папка
    _Глава NN.md             ← обзор главы + чек-лист подглав
    NN.1 ….md                ← подглава = файл
Приложения/                  ← глоссарий, шаблоны, литература, карта навыков
books/                       ← источники, только чтение (кроме Ключевые инсайты.md)
  Ключевые инсайты.md       ← единственный файл в books/, который можно дополнять
  инсайты/                   ← 23 тематических файла инсайтов (по темам: требования,
                               архитектура, аналитика, карьера и др.) — только чтение
  habr/                      ← статьи с Хабра по темам глав — только чтение
  telegram/                  ← tg_01–tg_05: материалы из Telegram-каналов — только чтение
_Входящие черновики/         ← сюда кладёшь сырые черновики из Word/Google Docs
```

Полная карта частей и глав — в `.claude/skills/book-coauthor/references/book-blueprint.md`.
**Перед советами по структуре сверяйся с живым состоянием волта**, а не только с картой: структура развивается, реальные папки — источник правды по подглавам.

`_Глава NN.md` — живой чек-лист подглав главы (чекбоксы). Обновляй его при добавлении или переименовании подглав.

Внутренние ссылки в волте — в формате Obsidian: `[[Папка/Файл|отображаемый текст]]`. Используй этот формат при создании перекрёстных ссылок между подглавами.

`books/инсайты/` содержит 23 тематических файла (только чтение):
аналитика-и-метрики, архитектура-и-интеграции, бизнес-процессы, взаимодействие-с-бизнесом,
видение-и-целеполагание, делегирование-и-процессы, инновации-и-изменения,
исполнение-и-ответственность, карьера-и-рост, качество-и-тестирование, командная-динамика,
лидерство-и-саморазвитие, методологии-и-фреймворки, найм-и-онбординг, организационная-политика,
переговоры-и-конфликты, психологическая-устойчивость, работа-с-людьми, разное, смена-роли,
стратегия-и-приоритизация, требования-и-спецификации, ясность-текста.

## Голос, стиль и режимы работы

Стиль книги, шаблон главы и правила цитирования вынесены в навык **`book-coauthor`**
(положи его в `.claude/skills/book-coauthor/`). Перед тем как писать или переписывать
любой связный текст главы — опирайся на этот навык. Кратко:

- **Голос:** живой пример из практики → теория простым языком → «как делать руками» → короткий итог. Личный опыт как вход, термины с пояснением в скобках, лёгкая ирония, вывод в конце.
- **Две аудитории:** новичок (основной поток) и практик (глубину выноси во врезки «для тех, кто уже в теме»).
- **Язык — русский.** Всегда.

## Правила работы (соблюдай всегда)

- **IMPORTANT: никогда не выдумывай личный опыт автора.** Где нужен реальный случай — ставь метку `[ПРИМЕР АВТОРА: нужен случай про …]` и спрашивай. Реальные кейсы — только от автора.
- **IMPORTANT: `books/` — только чтение** (кроме `books/Ключевые инсайты.md` — его можно дополнять). Не смешивай чужой текст с авторским голосом и не выдавай чужие слова за опыт автора. Атрибутируй явно («как пишет Вигерс…»). Подробно — в навыке, `references/citing-authors.md`.
- Не переписывай целые главы и соседние подглавы без явной просьбы. Сначала покажи, что и зачем меняешь.
- Не финализируй молча — это соавторство, последнее слово за автором.
- Не раздувай объём ради объёма: сказала подглава своё — заканчивай.

## Git и личные файлы

- Коммить структуру и `CLAUDE.md` — это общая память проекта.
- `books/` — в git (репозиторий приватный, автор работает с нескольких машин).
- `CLAUDE.local.md` держи вне git (`.gitignore`): личные заметки не коммитим.
