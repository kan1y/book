Итак что такое БД? 

представьте у вас есть продуктовый магазин, некий учет товаров вы ведете в лучшем случае в excel, а порой на практике чаще это некая тетрадка куда вы записываете поступление товара и его продажу клиентам, ну и какие долговые обязательства клиента перед магазином. Это все крайне неудобно. Тут вам поможет вести учет в базе данных, где у вас есть категория, виды товаров, цена, количество товара, в разных таблицах можно хранить некий учет поступление, отпуск товара, формирование задолженности и прочие вещи. Уже по этим данным можно структурировать асстортимент и получать нужные выгрузки, к примеру написать запрос и получить сколько пачек чипсов было куплено, и почему осталось 10, тем самым выявляя несоотвествия и тд

база данных это некий файл-хранилище, которое содержит в себе определенные данные. Наличие БД вам не даст зайти и посмотреть содержимое, для этого используется СУБД

СУБД - система управления базами данных. это программа которая позволяет вам работать с базами данных. СУБД PL/SQL Developer, Dbeaver, MSsql и др

Какие бывают виды БД, Есть два крупных разделения это реляционные и нереляционные. Помимо этого есть объектно ориентированные, колоночные и др

Реляционная БД - это база данных где имеется связи между таблицами, есть ключи (primary key первичный ключ) и foreign key (вторичный ключ) первый дает уникальный идентификатор позволяющий точно определить строку (id записи) второй дает возможность связывать с другими таблицами. Реляционные это oracle, postgre, mysql и др

нереляционные БД - это такая БД где нет связи, данные хранятся в виде объекта к примеру JSON

  

В банках чаще используется реляционные БД для хранения данных. В своей практике я работал с oracle, postgre, MSsql в работе, и с остальными видами для общего развития.

Давайте отпустим момент если вы с 0 решили в домашних ситуациях развернуть БД, представим что у вас первый рабочий день, вы открыли условную СУБД и начинаете работу.

Если в мире программирования начинается с классического Hello world, то в работе баз данных есть несколько опций это SELECT (выборка получение запросов) CREATE (создание) UPDATE (обновление) DELETE (удаление) INSERT (внесение изменений),

Есть такой термин как CRUD - create read update delete, который операции сверху исходят из него.

Для аналитика важнее всего в работе с БД будет опция выборки.

давайте представим у вас есть таблицы клиентов (clients), в которой есть поля фамилия, имя, дата рождения и статус клиента. 

1 давайте попробуем вытащить все данные из таблицы и так пишем

SELECT * FROM clients

получим некий список данных

Поздравляю это ваша первая выгрузка,

2 Давайте усложним условие добавим оператор WHERE (где) он задает условие, выберем только активных клиентов

SELECT * FROM clients where status = ‘active’

получаем выгрузку только клиентов имеющих активный статус

3 данные бывают выгружаются в неупорядочном  виде, давайте попробуем сделать сортировку, она делается за счет добавления в конце запроса Order by (дефолтно ASC если от от обратного то desc)

SELECT * FROM clients where status = ‘active’ order by lastname;

на выходе получили сортировку от А до Я по фамилии
