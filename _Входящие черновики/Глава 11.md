Гл. 11 — Интеграции  
Как системы разговаривают — для человека без бэкенд-опыта. Синхронный vs асинхронный режим. API на пальцах → REST → SOAP (где жив) → GraphQL кратко. JSON и XML. OpenAPI/Swagger. Брокеры: Kafka, RabbitMQ. Как аналитик описывает интеграцию в документе.  
Черновики сюда: объяснения API, примеры контрактов, интеграционные факапы, кейс монолит→микросервисы.

  

Бывает такое что в одной сфере начинает доминировать какой то определенный игрок, и банковская сфера Казахстана не исключение. Среди Банков Казахстана именно для физ лиц доминирующим по количеству пользователей является Kaspi банк, буквально на момент написания книги они добавили возможность оплаты через Apple Pay, очень долгое время банк был в своей закрытой экосистеме не давая другим игрокам какие либо доступы. Но позже некоторые банки, как и HCB, Forte, KMF и другие поняли это, что нет смысла конкурировать что легче подстроиться под текущей реалии и внедрить у себя интеграцию с другим банком. Банки подключились к QR каспи, тем самым возникает такой термин как API интеграция.

API - на собеседованиях ребята говорят стандартно application program interface, или супер простыми словами это чтобы приложение 1 понимало приложение 2.

Бывают несколько разновидностей API : REST SOAP GRaphQL

SOAP - мои любимые кандидаты зумеры про мыло говорят что это супер древняя разновидность, я каждый раз смеюсь с их ответов. так что же это такое , это строгий протокол представленный в виде XML формата, данные в сообщении передаютсся в виде тегов, имеется XSD схема где правила передачи формата, Для работы используется SOAP UI. Используется в банках, так к примеру ISO 20022 использует XML формат при обмене сообщений

[примеры XML, структура и тд]

REST - более современный архитектурный стиль. более гибкий, преимущественно JSON но и другие форматы. Программы чаще всего Postman. Swagger  для тестирования и просмотра методов. Тоже построен на принципах CRUD но только методы GET POST PUT PATCH DELETE

OPEN API это когда вы публикуете свои сервисы и другие компании подключаются к вам, получают client id client secret, scope и вызывают сервис. 

Брокеры сообщений нужны для передачи данных. Есть 2 крупных брокера кафка и раббит мк (кролик). Мне довелось поработать и с тем и с тем. Есть ряд отличий между этими 2 брокерами и поэтому используется разные. В банке о пополнений счета использовался кролик, в мобилках пуш уведомления используется кафка так как до 1 млн транзакций в секунду. 

  

Синхронный и ассинхронный вызов, синхронный можно привести пример что когда вы кому то звоните по телефону в этом момент человек параллельно с вами общается. Ассинхронный это принцип SMS вы отправили SMS и дальше идете по своим делам, как только собеседник прочитает сообщение то ответит в ответ.
