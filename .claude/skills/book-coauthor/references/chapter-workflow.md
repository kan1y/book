# Сборка главы из черновика (chapter-workflow)

> Режим по команде: «оформи в главу», «собери из черновика», «доведи подглаву», «причеши по шаблону».
> Цель — превратить сырой MD-черновик автора в готовый текст в его голосе, не подменяя смысл.

## Перед началом
1. Прочитай `voice-guide.md` (голос) и `book-blueprint.md` (место главы в книге, аудитория).
2. Прочитай сам черновик и **соседние подглавы** в той же папке-главе — чтобы не дублировать и не противоречить.
3. Если в черновике есть отсылка к другому автору — НЕ выдумывай цитату; работай по `citing-authors.md`.

## Процесс
1. **Пойми замысел.** Что автор хочет сказать этой подглавой? Если неясно — спроси одним вопросом, не угадывай.
2. **Собери спину:** где живой пример, где теория, где «как делать руками», где итог. Если в черновике нет авторского примера — используй обобщённый: опыт условного коллеги («один знакомый аналитик…»), известный отраслевой кейс, ссылку на книгу. Выдуманный личный опыт автора и присвоение ему чужих историй — под запретом.
3. **Разверни в голосе автора** по шаблону ниже.
4. **Покажи, что изменил.** Кратко: что добавил, что переписал, где оставил [заглушку для примера автора].
5. **Не финализируй молча.** Это соавторство — оставь автору правки и выбор.

## Единый шаблон подглавы

```markdown
# [Название подглавы]

<!-- ВХОД: живая сцена из практики, 1–2 абзаца. Цепляет и ставит проблему. -->

<!-- СУТЬ: теория простым языком. Термины — с пояснением в скобках при первом появлении.
     Метафора из быта, если тема абстрактная. Списки — только где реально помогают. -->

<!-- КАК ДЕЛАТЬ РУКАМИ: конкретика. Пример, шаги, ДО/ПОСЛЕ, мини-чек-лист.
     Здесь читатель получает то, что можно применить завтра. -->

<!-- ДЛЯ ТЕХ, КТО УЖЕ В ТЕМЕ (опционально): врезка с глубиной для практика. -->
> **Для тех, кто уже в теме:** […]

<!-- ИТОГ: 2–4 предложения. Что запомнить / что сделать / к чему ведёт. -->
```

## Метки для автора
Используй явные метки-заглушки, чтобы автор видел, где нужен его вклад:
- `[ПРИМЕР АВТОРА: нужен случай из практики про …]`
- `[ПРОВЕРИТЬ ФАКТ: …]`
- `[ОТСЫЛКА К АВТОРУ: см. citing-authors.md, тема …]`

Приоритет при работе с примерами:
1. Автор написал личный кейс в черновике → используй его.
2. Кейса нет → замени обобщённым (коллега, отраслевой случай, ссылка на книгу) без присвоения автору.
3. Обобщённый не подходит (место требует именно личного голоса) → оставь метку с объяснением: `[ПРИМЕР АВТОРА: нужен случай про … — обобщённый здесь не подходит, потому что …]`.

## Что не делать
- Не превращать живой текст в учебник нейтральным голосом.
- Не раздувать объём ради объёма — если подглава сказала своё, заканчивай.
- Не переписывать соседние подглавы заодно без просьбы.
