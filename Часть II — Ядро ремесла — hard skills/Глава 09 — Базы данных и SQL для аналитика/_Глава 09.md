# Глава 09. Базы данных и SQL для аналитика

_Часть II. Ядро ремесла — hard skills_

Ключевая и объёмная глава: путь от новичка («никогда не открывал базу») до продвинутого уровня («понимаю, о чём спорят архитекторы»). Сквозная идея — «не стать DBA, а читать и понимать».

## Подглавы

- [x] 09.1 Зачем аналитику базы данных — вступление, личный опыт (PL/SQL-команда), метафора магазина, охват главы
- [x] 09.2 БД, СУБД и внутреннее устройство — БД vs СУБД, инструменты, иерархия, типы данных
- [x] 09.3 Реляционные и нереляционные БД — PK/FK, семейства, NoSQL (Mongo/Redis/колоночные), когда что
- [x] 09.4 SQL с нуля — SELECT, WHERE, ORDER BY — CRUD, первая выборка
- [x] 09.5 JOIN — соединяем таблицы — INNER/LEFT/RIGHT/FULL
- [x] 09.6 Группировка, объединение и подзапросы — GROUP BY, HAVING, агрегаты, UNION, CASE, подзапросы, CTE, порядок выполнения
- [x] 09.7 Процедуры, функции, сессии и джобы — «движок» БД и как его читать
- [x] 09.8 Продвинутый уровень — транзакции/ACID, индексы, нормализация, масштабирование (необязательная глубина)
- [x] 09.9 Как читать чужую схему данных — реверс-инжиниринг, вопросы, права и осторожность + итог главы

## Опора на источники (books/)
- Алан Бьюли «Изучаем SQL» — подход «учись руками», SELECT (09.4)
- Кристофер Дейт «SQL и реляционная теория» — реляционная модель, ловушка NULL (09.3)
- Энтони Молинаро «SQL: сборник рецептов» — агрегация, оконные функции (09.6)
- Егор Рогов «PostgreSQL 17 изнутри» — индексы, EXPLAIN (09.8)
- Мартин Клеппман «Высоконагруженные приложения» — репликация/шардирование, CAP, OLAP/OLTP (09.8)
- MongoDB. Полное руководство (Брэдшоу, Ходоров) — документная модель (09.3)
